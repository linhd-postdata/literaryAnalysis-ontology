postdata-literaryAnalysisElements ontology: Description back to ToC

As mentioned in previously sections, the objective of this ontology is to cover the aspects relating with the literary analysis of a Redaction of a Poetic Work in the postdata-core ontology .
The ontology prefix is pdliteraryAnalysis

Therefore following classes were modeled:

  • The pdliteraryAnalysis:Acrostic class, which is grouping of the first letter of each line that form a word, the alphabet or any other message. This class also includes telestichs, that is, the message create it with the last letter of each line. This class is an specialization of frbroo:Expression
  • The pdliteraryAnalysis:Intertextuality class, which is shared textual materials between two or more contemporary texts. It is a Text from which information or ideas are derived for the composition of the PoeticWork/Redaction or it is a text that took information or ideas from the PoeticWork/Redaction. This class is also an specialization of frbroo:Expression
  • The pdliteraryAnalysis:FigureOfSpeech class, a word or phrase used in a non-literal sense for rhetorical or vivid effect. This is a specialization of raop:RethoricalDevice class and is a basis of a hierarchy formed by the following classes:
    • Trope class, that represents a figure of speech which the use of a word or a phrase with a meaning that is not the usual one, like Metaphor or Hyperbole.
    • Scheme class in which changing in the usual order of words is produced like Alliteration or Enjambment

In this ontology, rhetorical figures have particular relevance. I have taken into account the possibility of showing the words that intervene in the figure using a pdliteraryAnalysis:content property and its description with the dc:description property, both generic properties that can be applied to many classes.
On the other hand and as specific properties of these figures, we have considered the properties pdliteraryAnalysis:figureMeaning, to represent the meaning of the figure and the property pdliteraryAnalysis:figureReferent for an unspoken name which is equivalent to the figure. All these properties are data properties, and their range is a String.
Among the properties of objects related to the figures, we must highlight the property isPresentIn that refers to the lines in which the figure appears. In this case, you should use elements from the postdata-structuralElements ontology.Other important properties related to the figures are the following:

  • pdliteraryAnalysis:rethoricalOperation whose range is a controlled vocabulary pdskos:rethorical-operation .
  • pdliteraryAnalysis:figureVariety which is defined with generic semantics and therefore specialises specifically for tropeVariety and schemeVariety. these last two properties have as a range the controlled vocabulary pdskos:trope-variety and pdskos:scheme-variety respectively.
